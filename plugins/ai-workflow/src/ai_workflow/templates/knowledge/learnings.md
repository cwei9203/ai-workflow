# Durable Learnings

This file contains project-specific lessons promoted from completed task
evidence. It starts empty by design. Do not add generic coding advice or
unverified observations.

## Entry format

Use one stable identifier per lesson:

```markdown
## L-001: Imperative rule or concise finding

- Status: active
- Scope: path, subsystem, or project-wide
- Source tasks: TASK-YYYYMMDD-HHMMSS, TASK-YYYYMMDD-HHMMSS
- Last verified: YYYY-MM-DD

State what a future agent must know or do, when it applies, and the failure it
prevents. Link to authoritative code, tests, or project documentation where
useful.
```

Allowed status values are `active` and `superseded`. A superseded entry must
point to its replacement or explain why it no longer applies. When repeated
tasks reinforce a lesson, update its source-task list rather than duplicating
the entry.
